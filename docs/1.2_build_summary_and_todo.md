# Seekspot Build Summary & Local Dev Setup

## seekspot-frontend (20 source files + 5 config files)

- Vue 3 + Vite (base: `/seekspot/`), Tailwind, vue-i18n (zh/en)
- **3 npm scripts:** `npm run dev` (localhost:5174), `npm run build:sandbox`, `npm run preview` (serves built `dist/` locally to smoke-test before deploying)
- **8 routes total:** 5 main pages + `/login`, `/register`, `/verify`
  - `/qa`, `/online-application`, `/submit-documents` redirect to `/login` if unauthenticated
  - `/` and `/dashboard` are public
- **Register form** collects: username, email, password, phone number, user type (individual / agent / school radio)
  - Client-side validation: username 3–30 chars (alphanumeric, `_`, `-`); password min 8 chars; phone required + format checked
- **Login** accepts email or username as the identifier; stores both `token` and `user_type` in localStorage on success
- **Origin header** (`x-seekspot-origin`): injected on all axios requests via a global interceptor in `main.js`; the value compiles to empty string in dev (header is not sent) and `seekspot_sandbox_origin_key` in sandbox builds

## seekspot-backend (10 source files + 4 root files)

- Node.js + Express 5.x, ES Modules (`"type": "module"`)
- `seekspotdb` with user types: `individual`, `agent`, `administrator`, `school`
  - Public registration accepts only `individual`, `agent`, `school` — `administrator` is intentionally excluded and must be inserted directly into the DB
- JWT payload keys: `userId` + `userType` (camelCase); login response also returns `user_type` (snake_case) for frontend localStorage storage
- **2 docker-compose files:** local dev (spins up MySQL container on host port 3307 to avoid seekschool conflict) and sandbox (connects to CVM MySQL — no container)
- Migration SQL at `src/database/migrations/001_create_seekspotdb.sql` — run once on CVM MySQL before first sandbox deployment

## Next Steps — Run Locally

### Prerequisites

- Docker Desktop installed and running
- Node.js installed (for the frontend)

---

### Step 1 — Create the backend environment file (Done)

```bash
cd seekspot-backend
cp .env.dev.example .env.dev
```

Open `.env.dev` and verify the values. The defaults in the example are ready for local dev — no changes needed unless you want different DB credentials.

---

### Step 2 — Start the backend (Done)

```bash
cd seekspot-backend
docker compose --env-file .env.dev up -d
```

> The `--env-file .env.dev` flag is required. The MySQL container uses `${MYSQL_DATABASE}` etc. for its own init — docker-compose needs the flag to substitute those from `.env.dev` (it does not auto-discover `.env.dev` by name).

This starts:
- A **MySQL 8** container on host port **3307**
- The **Express backend** on port **3001**, using `wait-on` to hold until MySQL is ready

First run pulls the MySQL image — takes a minute. When ready you'll see:

```
Seekspot backend running on port 3001
```

> **Dev mode behaviour:** `NODE_ENV=development` skips email verification entirely. Registration creates the user immediately and logs the verification code to the console. The Register page detects the `DEV_SKIP_VERIFICATION` response, auto-calls login, and redirects to `/` — no email step.

---

### Step 3 — Start the frontend (Done)

Open a new terminal:

```bash
cd seekspot-frontend
npm install        # first time only
npm run dev
```

Frontend runs at: **http://localhost:5174/seekspot/**

---

### Step 4 — Verify (Done and tested)

1. Open **http://localhost:5174/seekspot/** — Home page loads
2. **Register:** fill all fields → submit → auto-logged in and redirected to `/` (dev mode, no email)
3. **Login:** accepts email or username as the identifier
4. **Auth guard:** navigate to `/qa` while logged out → redirects to `/login`

## Day-to-Day Dev Workflow

| Task | Command |
|---|---|
| Start backend | `docker compose --env-file .env.dev up -d` (from `seekspot-backend/`) |
| Start frontend | `npm run dev` (from `seekspot-frontend/`) |
| Stop backend | `docker compose down` |
| Reset DB entirely | `docker compose down -v` |
| Frontend changes | Hot reload — Vite watches automatically |
| Backend changes | **No hot reload in Docker** (uses `node`, not `nodemon`) — i can just do docker compose down and up |
| Smoke-test a production build | `npm run build:sandbox` then `npm run preview` |
